Map Input Tracker

Design and Development of Map Input Tracker API. RESTful developer API to be integrated with Platforms for getting Map Inputs.

FLOW

Workflow Management Platform, specifically built to Automate the Orchestration within different Map-Making Processes.

It offers the following features to the users:

  • ▪ Offers a platform to customize map-making processes and easily deploy them (without making any changes to the code).
  • ▪ Automatically creates Tasks and Routes them across different Teams involved in the Map-Making Process.
  • ▪ Provides a pre-built Programmatic Interface to communicate with MCP and SO Systems.
  • ▪ Provides dashboards for real-time tracking and analytics of BI Reports.

LeadStore

Capture, Track and Analyze all Map Observations.

  • ▪ Standard way to Create, Query and Use Leads across all Lead Types.
  • ▪ Provide a platform to Store Observations* and Sightings*.
  • ▪ Provide a platform to Store Leads for processes that do not have a Store of Their Own.
  • ▪ Bring all Existing LeadStores under Common Platform with eventual goal of Single Unified LeadStore.

*Lead: Any input for a map update with sufficient information to process further for content creation.
*Sighting: Sighting is a source which has the potential of becoming lead. A sighting might be a lead with very low confidence or it may not have all the required fields of lead.
